# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.0.x   | Yes       |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

To report a vulnerability, send an email to **info@memora.ca** with:

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested remediation (optional)

Please encrypt sensitive details using our PGP key if available, or note that you require a secure channel before sharing details.

## Response Timeline

- **48 hours**: acknowledgment of your report
- **7 days**: confirmation of the vulnerability and a fix plan
- **30 days**: targeted resolution for confirmed critical issues

We will keep you informed throughout the process. If a timeline cannot be met, we will notify you with an updated estimate.

## Responsible Disclosure

We ask that you:

- Give us reasonable time to investigate and patch before any public disclosure
- Avoid accessing, modifying, or deleting data that is not your own
- Not perform denial-of-service testing or social engineering attacks

We commit to:

- Treat your report with confidentiality
- Credit you in the release notes (unless you prefer to remain anonymous)
- Not pursue legal action for good-faith security research

Thank you for helping keep this project and its users secure.
